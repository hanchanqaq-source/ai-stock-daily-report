# 股票基金双中心 Build D3：行业周期与经营生产力代理证据

## 原因

Build D2 已回答基金“主要投什么、集中度多高、和其他基金披露持仓是否重复”，但不能据此直接声称行业处于某个周期，也不能把短期行情上涨误写成生产力改善。

Build D3 因此只补充可解释的证据层：短期行业周期和较长期经营生产力代理分开计算、分开标注日期与置信度。当前用户组合风险、配置和调仓建议仍属于 Build D4。

## 数据与映射边界

公开数据继续使用 AKShare：

- `fund_portfolio_industry_allocation_em`：基金披露行业及权重；
- `fund_portfolio_hold_em`：最新披露前十大持仓及报告期；
- `stock_board_industry_name_em`：东方财富行业板块目录；
- `stock_board_industry_hist_em`：行业板块历史收盘和成交额；
- `stock_board_industry_cons_em`：行业成份、当日涨跌幅、动态市盈率和市净率；
- `index_zh_a_hist`：沪深300（`000300`）历史行情；
- `stock_yjbb_em`：指定报告期的营收同比、利润同比、ROE、毛利率和每股经营现金流。

行业连接只允许两条可审计路径：

1. 基金披露行业名称与 AKShare 行业板块名称完全一致；
2. 最新披露前十大持仓代码在 AKShare 业绩报表中有明确“所处行业”，且该行业存在于板块目录。

系统不按基金名称、证券名称、相邻赛道或关键词猜行业。每只基金最多选择 3 个可核验行业，同次请求按基金披露权重合计最多分析 6 个行业；被省略数量和未分类持仓必须显示。

## 周期证据与阶段

周期模块只使用以下市场证据：

- 行业 20 日收益；
- 行业 60 日收益；
- 最近 20 日平均成交额相对此前 20 日的变化；
- 当前成份股上涨比例；
- 行业 20 日收益相对沪深300的差值。

至少需要 4 项证据，且 20/60 日收益必须存在，否则阶段固定为 `insufficient`（证据不足）。阶段由固定阈值生成：

| 阶段 | 主要规则 |
| --- | --- |
| 过热 `overheated` | 20 日 > 8%、60 日 > 12%、上涨比例 ≥ 70%、成交额变化 > 25% |
| 扩张 `expansion` | 综合分 ≥ 3，且 20/60 日均为正 |
| 收缩 `contraction` | 综合分 ≤ -3，且 20/60 日均为负 |
| 放缓 `slowdown` | 20 日为负，且 60 日仍为正或综合分 ≤ -1 |
| 复苏 `recovery` | 20 日非负且综合分 ≥ 1 |
| 证据不足 `insufficient` | 样本不足、证据冲突或不满足上述可解释条件 |

综合分只采用文档列出的固定门槛，不使用 AI，不训练模型，不读取 Microsoft Qlib。Qlib 仅作为“特征分层、状态判断、结果评估必须可复现”的方法参考，不作为依赖打入便携版。

动态市盈率和市净率只显示当前成份股正值中位数；没有历史分位时不得称为“估值高位/低位”，也不进入阶段评分。

## 经营生产力代理

本模块不声称计算全要素生产率。它只把最新可用业绩报告中的下列横截面统计称为“经营生产力代理证据”：

- 营收同比中位数；
- 净利润同比中位数；
- ROE 中位数；
- 毛利率中位数；
- 每股经营现金流为正的成份股比例。

覆盖不足或核心指标少于 3 项时返回 `insufficient`。覆盖满足要求后，以营收、利润、ROE 和经营现金流固定门槛形成 `improving`、`stable` 或 `weakening`。资本开支、库存周期、渗透率、订单和政策没有统一可比数据时必须列为缺失，不能用行情替代。

## 页面和请求

- 页面：`/funds/industry-cycle`；
- 支持 1–4 个不重复六位基金代码；
- 每次请求必须重新人工确认；
- 页面初次渲染不自动请求；
- 结果只保存在 React 内存，刷新清空；
- API：`POST /api/v1/provider-readonly/akshare/funds/industry-cycle`；
- 固定 sections：`funds`、`disclosed-holdings`、`industry-cycle-evidence`、`productivity-proxy-evidence`。

## 安全与失败语义

- API 仅允许 localhost；
- `readOnly` 和 `humanApproved` 必须为 `true`；
- 账户、交易、通知、AI、持久化能力必须全部为 `false`；
- 固定超时，错误只返回低敏错误码，不透传上游异常；
- 单个行业行情或成份接口失败时降级为缺失/证据不足，不伪造数据；
- 测试只使用注入的 Fake AKShare，不访问真实网络；
- 不读取、输出或修改 `.env`、密钥、Webhook 或系统环境；
- 不接 Qlib 整包、不启动数据库、不生成买卖、仓位、调仓或自动执行建议。

## 回滚

回滚 D3 时只移除 `src/fund_industry_cycle.py`、D3 只读 API、行业周期页面组件、对应测试和本文档；D1/D2 的基金资料、净值、持仓、比较、行业穿透，以及股票中心和 Windows Portable-M1 均不受影响。
