# Web mock-only 页面阶段收口说明

本文档用于收口“股票基金质量分析系统”的 Web mock-only 页面阶段，固定日报 / 推送显示名称为“AI股票基金每日信息报告”。当前内容仅用于确认本地静态预览页面的结构、文案、安全边界与后续阶段入口，不代表真实业务能力已经接入。

## 1. 阶段定位

当前阶段是 Windows 本地 mock-only 页面预览阶段，仅用于本地 UI 渲染检查。页面只允许使用静态脱敏 fixture，不代表正式日报，不代表真实账户分析，不构成投资建议，不发送任何通知，也不执行任何交易。

本阶段的核心边界如下：

- 仅服务于 Windows 本地 mock-only 页面预览。
- 仅面向本地 UI 渲染、布局、文案和模块完整性检查。
- 仅使用静态脱敏 fixture，不读取真实账户、真实基金平台、真实历史日报、数据库或用户本地文件。
- 不代表“AI股票基金每日信息报告”的正式日报内容。
- 不代表任何真实账户、持仓、基金或交易分析结果。
- 不构成投资建议、收益承诺、买卖指令或自动化执行依据。
- 不启动后端，不连接真实 API，不接入 provider，不调用 AI / Agent / 模型服务。
- 不发送日报、提醒、Webhook、Bot、邮件或任何外部通知。
- 不执行交易，不写入本地配置，不上传日志。

## 2. 已完成模块清单

- **Web-P21 仪表盘摘要 mock-only**：新增 mock-only 仪表盘摘要预览，用静态脱敏指标展示页面顶部摘要能力。该模块只用于观察 UI 卡片、状态文案和风险提示，不读取真实行情或账户数据。
- **Web-P22 持仓预览 mock-only**：新增持仓预览模块，用脱敏 fixture 展示账户概览、持仓列表、观察备注和非投资建议提示。该模块不代表真实持仓，也不会触发交易或账户读取。
- **Web-P23 历史报告预览 mock-only**：新增历史报告预览模块，用静态样例展示报告概览、列表和详情结构。该模块不读取真实历史日报文件，也不生成正式日报。
- **Web-P24 提醒预览 mock-only**：新增提醒预览模块，用脱敏样例展示提醒规则、触发记录和发送状态。该模块只展示 UI，不发送提醒、Webhook、Bot 或邮件。
- **Web-P25 Agent 对话预览 mock-only**：新增 Agent 对话预览模块，用静态会话、消息、流式片段和错误示例展示未来交互形态。该模块不启动真实 Agent，不调用 OpenAI、DeepSeek、智谱、本地大模型或其他模型 provider。
- **Web-P26 空状态与错误示例 mock-only**：新增空状态、错误状态和降级状态示例，便于检查无数据、失败和受限场景的 UI 表达。该模块不通过真实请求制造错误，也不上传日志。
- **Web-P27 页面导航与返回顶部优化 mock-only**：优化长页面浏览体验，补充页面快速导航、稳定锚点、返回顶部和返回模块列表入口。该能力只改变静态页面导航体验，不新增数据能力。
- **Web-P28 页面总览 / 模块完成度状态栏 mock-only**：新增页面总览与模块完成度状态栏，仅从真正业务预览模块派生可预览、后续建设和总模块数量；安全边界、设置与导入导出属于页面基础设施，不纳入业务模块完成度统计。该状态栏同时强化 REDACTED FIXTURE DATA、127.0.0.1 only、无网络、无通知、无交易、无模型调用等边界。
- **Web-P29 移动端 / 窄屏显示优化 mock-only**：优化 mock-only 页面在平板和手机宽度下的响应式布局，使页面总览、快速导航、模块卡片和返回入口更适合窄屏检查。该优化仅影响展示布局，不新增真实数据读取或运行时能力。

### Web-P30.1 数据脱敏与完成度口径补充

- mock-only 持仓 fixture 使用明显虚构金额、比例和收益率，例如 `¥88,888.88`、`¥200,000.00`、`44.4%`、`¥20,000.00`，不得复制用户真实持仓金额、收益率、基金代码、交易记录、通知目标或密钥。
- 页面文案必须持续标明 `模拟数据`、`REDACTED FIXTURE DATA`、`非真实账户`、`非投资建议`、`不会发送通知`、`不会交易`。
- 安全边界和本地设置与导入导出预览是页面基础设施；页面总览中的完成度只统计 6 个业务预览模块，不用基础设施状态稀释业务模块完成度。

## 4. 本地验证方法

Windows 本地验证流程如下：

1. 下载最新 `main` 分支 ZIP。
2. 删除旧目录：

   ```cmd
   rmdir /s /q D:\AI_Workspace\ai-stock-daily-report-main
   ```

3. 重新解压 ZIP 到 `D:\AI_Workspace\ai-stock-daily-report-main`。
4. 打开 CMD，进入项目目录：

   ```cmd
   cd /d D:\AI_Workspace\ai-stock-daily-report-main
   ```

5. 运行一键安全预览脚本：

   ```cmd
   scripts\windows_localhost_web_safe_preview_one_click.bat
   ```

6. 手动打开浏览器访问：

   ```text
   http://127.0.0.1:5174/mock-only-preview/
   ```

成功时，页面应能看到以下关键文字或区域：

- `MOCK ONLY`
- `LOCAL PREVIEW ONLY`
- `NO REAL NETWORK`
- `NO REAL ACCOUNT`
- `NO OUTBOUND DELIVERY`
- `AI股票基金每日信息报告`
- `股票基金质量分析系统`
- 页面总览
- 页面快速导航
- 各 mock-only 预览模块

## 6. 安全边界总表

| 分类 | 允许 | 禁止 |
| --- | --- | --- |
| 页面与运行方式 | mock-only 页面渲染；`127.0.0.1` 本地预览；本地 UI 检查；本地文档阅读 | 绑定或要求使用 `0.0.0.0`；自动打开浏览器；启动后端；生产部署 |
| 数据来源 | 静态脱敏 fixture | 真实 API；真实 provider；真实账户读取；真实基金平台读取；真实历史日报文件读取；数据库读取；用户本地文件读取 |
| 模型与 Agent | 静态 Agent 对话样例展示 | OpenAI；DeepSeek；智谱；本地大模型；LangChain；model provider SDK；真实 Agent |
| 通知与外联 | 本地静态通知状态展示 | 通知发送；日报发送；提醒发送；上传日志；外部网络请求；Webhook / Bot / 邮件真实发送 |
| 配置与密钥 | 文档中的安全边界说明 | `.env` 读取；token / webhook / API key 读取；写入本地配置；写入 `localStorage` / `sessionStorage` |
| 浏览器能力 | 静态 HTML / CSS / mock-only JavaScript 渲染 | `fetch`；`axios`；`XMLHttpRequest`；`WebSocket`；`EventSource`；`localStorage`；`sessionStorage`；`indexedDB`；Notification API；Service Worker；`navigator.sendBeacon`；`FileReader`；`input type="file"` |
| 交易与执行 | 非投资建议提示；静态交易禁止提示 | 自动交易；手动交易代办；买卖指令；收益承诺；任何真实下单能力 |

## 8. 下一阶段建议

下一阶段可以考虑进入数据结构准备，但建议继续拆分为小步：

- **Web-P31 日报 fixture 结构统一**：梳理 mock-only 页面所需 fixture 的统一目录、命名和字段边界。
- **Web-P32 mock 报告 schema 文档**：记录 mock 报告的字段含义、脱敏规则、展示约束和未来兼容点。
- **Web-P33 本地 mock 数据目录规划**：明确 mock 数据目录、只读边界、禁止真实数据混入的检查方式。
- **Web-P34 前端预览字段与未来真实日报字段对齐**：只做字段语义对齐和文档约束，不接真实 API、provider、AI、通知或账户。

进入 Web-P31 前，建议先做一次高规格总复核，确认 mock-only 阶段安全边界、页面结构、文档和测试都一致，再进入数据结构准备阶段。复核应重点确认：页面仍然 mock-only、脚本仍然只绑定 `127.0.0.1`、没有真实网络能力、没有密钥读取、没有真实 Agent / AI / 通知 / provider / 账户 / 交易入口。

## 9. 回滚说明

如果页面验证失败，可以先回退到最近一个已知稳定的 `main` 版本，再重新执行 Windows 本地验证流程。也可以优先检查最近 PR 中与 mock-only HTML、CSS、entry、Vite preview 配置或 Windows one-click 脚本相关的变更，确认是否引入了布局、入口路径或安全边界回归。

回滚或排查时仍需保持以下原则：不接真实 API，不启动后端，不读取 `.env`，不读取 token / webhook / API key，不调用模型，不发送通知，不执行交易。
