# 美股指数矩阵与广度指标说明

## 1. 设计目标

本模块用于细化美股指数矩阵和广度 / 体感指标计算框架，承接 P5-L0 的全球市场页面切换模型，并为后续真实数据源接入预留稳定结构。

当前阶段只处理调用方传入的样例数据，不联网、不抓真实行情、不接入 yfinance / Yahoo Finance / TradingView / Nasdaq / NYSE / Polygon / Alpha Vantage，也不保存真实价格、真实涨跌幅或真实成交额。

## 2. 美股官方指数矩阵

美股页面按以下分组组织：

- 权重核心：标普500、道琼斯工业平均指数、纳斯达克综合指数。
- 科技成长：纳斯达克100、费城半导体指数。
- 中小盘：罗素2000、标普400、标普600。
- 等权 / 广度：标普500等权指数，以及系统计算的广度 / 体感指标。

其中标普500、道琼斯工业平均指数、纳斯达克综合指数、纳斯达克100、费城半导体指数、罗素2000、标普400、标普600、标普500等权指数均为 `official_index`。

## 3. 美股广度 / 体感指标

当前支持的系统计算指标包括：

- 美股中位数涨跌幅。
- 美股平均涨跌幅。
- 美股上涨家数占比。
- 美股全市场等权涨跌。
- NYSE上涨家数占比。
- Nasdaq上涨家数占比。

这些指标用于观察输入样本的市场体感和广度结构，不输出买卖建议，不代表官方指数。

## 4. 官方指数 vs 系统计算指标

`official_index` 表示公开指数项；当前阶段只定义矩阵结构和后续数据源需求，不写入行情数值。

`computed_indicator` 表示系统根据输入股票数据计算的体感指标，非官方指数。所有美股体感指标必须设置 `item_type=computed_indicator`、`is_computed=true`、`is_official=false`，并在 `disclaimer` 中说明“非官方指数”。

## 5. 计算口径

- 中位数使用所有有效 `change_pct` 的中位数。
- 平均涨跌使用所有有效 `change_pct` 的算术平均值。
- 上涨家数占比使用 `change_pct > 0` 的数量 / 有效 `change_pct` 数量。
- 全市场等权涨跌当前使用有效股票涨跌幅的简单平均作为近似值，后续接入正式数据源后可优化口径。
- NYSE上涨家数占比只统计 `exchange=NYSE` 且 `change_pct` 有效的样本。
- Nasdaq上涨家数占比只统计 `exchange=NASDAQ` / `exchange=Nasdaq` 且 `change_pct` 有效的样本。
- `None`、空字符串和非数字 `change_pct` 会被忽略。

## 6. 美股和 A 股差异

美股没有 A 股那种统一的涨跌停制度。本阶段不设置美股涨跌停差为核心体感指标；如结构中需要表达该字段，仅使用 `limit_up_down_diff_status=not_applicable`，不照搬 A 股涨跌停差逻辑。

## 7. 数据限制

本阶段不联网、不抓真实行情，只处理输入数据。输出结构的 `data_mode` 为 `model_only` 或 `computed_from_input`，并保持 `has_realtime_data=false`。
