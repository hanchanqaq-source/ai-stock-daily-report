# 股票基金双中心 Build D4 当前用户基金组合风险与配置建议

## 1. 原因

Build D1—D3 已提供基金公开事实、基金对比、行业穿透、行业周期和经营生产力代理证据，但仍缺少当前用户组合视角。Build D4 将这些证据与当前页面内存中的基金持仓组合，形成可解释的组合风险和配置复核建议。

## 2. 处理

- 当前用户全部基金持仓用于计算基金组合内部权重、单只集中度、前三集中度和用户目标偏离；
- 相同代码的多条持仓在本次计算中合并，不写回、不持久化；
- 最多接收 20 条持仓；按金额选择前 4 个基金代码补充 D2/D3 公开证据，并显示公开证据覆盖率；
- 金额和盈亏只进入本机 `127.0.0.1` 后端，不传给 AKShare；AKShare 只收到基金代码；
- 固定阈值识别基金类型集中、披露持仓/行业重合、周期逆风、经营代理走弱和目标偏离；
- 缺失行业、周期、生产力或覆盖率不足时返回“证据不足/部分”，不得当作低风险；
- 建议仅包含人工复核和按用户自定目标再平衡，不包含具体买卖单、收益承诺或自动执行。

## 4. 固定风险阈值

- 单只基金占比 ≥35%：高风险复核；25%—35%：中风险关注；
- 前三只合计 ≥75% 且组合超过 3 只：中风险复核；
- 已核验基金类型占组合 ≥60%：中风险复核；
- 披露持仓重合下限 ≥20% 或披露行业重合下限 ≥30%：中风险；更高阈值升级为高风险；
- 用户目标偏离绝对值 ≥5 个百分点：提示人工再平衡复核；
- 公开证据覆盖低于 70%：必须提示覆盖不足。

阈值只是透明、固定、可测试的审慎规则，不代表适合所有人的投资结论。

## 5. 安全与数据边界

- 每次必须由用户勾选确认后运行，不自动读取；
- 仅接受六位基金代码和当前页面内存持仓；刷新后持仓仍会清空；
- 不读取 `.env`、密钥、真实账户或券商数据；
- 不调用 AI、不发送通知、不交易、不保存；
- 公开披露不是实时完整仓位，周期阶段不是交易信号，经营生产力为代理证据而非长期收益预测；
- 组合只包含基金，不包含现金、股票、负债和外部账户。

## 6. 回滚

只回滚 D4 风险引擎、只读 API、基金建议页面、测试和本文档；D1—D3、股票中心、用户隔离和 Windows Portable-M1 不回滚。
