# 股票基金双中心 Build D4：当前用户基金组合风险与配置复核

## 原因

Build D1–D3 已提供基金资料、正式净值、最新披露持仓、基金披露行业、基金间重合下限、行业周期和经营生产力代理证据，但这些页面仍以单只基金或手动输入代码为中心，不能回答“当前选中用户的基金组合风险集中在哪里、与自己记录的目标仓位是否偏离”。

Build D4 将当前用户的内存基金持仓与公开证据连接起来，并把风险事实、固定阈值、配置复核提示和缺失项分开展示。它不读取账户，不生成强制买卖指令，不自动调仓。

## 当前用户与隐私数据流

1. 页面只读取 `PortfolioUserContext.activeFundHoldings`；股票持仓不进入 D4。
2. 浏览器根据正数持有金额计算组合占比，相同六位基金代码先合并，每次最多 4 只基金。
3. 发送到本机只读 API 的持仓字段固定为：
   - 六位基金代码；
   - 归一化占比；
   - 用户自行填写的目标仓位，未填写时为 `null`。
4. 用户名、用户 ID、持有金额、收益、成本、账户和备注都不进入请求。
5. 切换用户、修改持仓或刷新页面后，确认状态和分析结果立即清空。
6. 结果只存在 React 内存中；Build E 才处理持久化、迁移、备份和回滚。

API：`POST /api/v1/provider-readonly/akshare/funds/portfolio-advice`。

固定 sections：`portfolio-allocation`、`nav-risk`、`disclosed-overlap`、`industry-cycle`、`allocation-guidance`。

## 五个风险证据维度

### 1. 基金集中度

- 最大单只基金占比；
- 前两只基金占比；
- Herfindahl 指数；
- 有效基金数量 `1 / HHI`。

稳健、均衡、进取只改变固定关注阈值，不代表系统替用户完成风险测评，也不形成推荐比例：

| 风险偏好 | 单只基金关注阈值 | 前两只关注阈值 |
| --- | ---: | ---: |
| 稳健 | 30% | 60% |
| 均衡 | 40% | 70% |
| 进取 | 50% | 80% |

超过阈值为 `watch`，超过阈值 10 个百分点为 `high`。

### 2. 披露重合

- 复用 D2 最新披露前十大持仓重合下限；
- 复用 D2 基金披露行业重合；
- 1 只基金时标记为不适用；
- 最大重合超过 15% 为关注，超过 30% 为优先复核。

重合只代表公开披露样本下限，不代表完整、实时组合重合度。

### 3. 披露行业集中

组合行业暴露按 `组合基金占比 × 基金披露行业占净值比例` 汇总，显示前五行业、前三行业合计、公开披露覆盖率和未披露/未分类比例。未知部分保持未知。

| 风险偏好 | 单一披露行业关注阈值 |
| --- | ---: |
| 稳健 | 20% |
| 均衡 | 25% |
| 进取 | 35% |

该结果不是实时全持仓行业穿透。

### 4. 净值波动与回撤

每只基金使用 AKShare 正式单位净值序列计算：

- 20 日收益；
- 60 日收益；
- 最近 60 个日收益的年化波动率；
- 最近最多 120 个净值点的最大回撤。

组合摘要中的“加权平均基金波动”是各基金波动率按当前占比加权，不包含基金间协方差，不能称为完整组合波动率。回撤摘要显示各基金样本中的最差回撤。

| 风险偏好 | 年化波动关注阈值 | 回撤幅度关注阈值 |
| --- | ---: | ---: |
| 稳健 | 18% | 10% |
| 均衡 | 25% | 15% |
| 进取 | 35% | 25% |

### 5. 行业周期压力

复用 D3 最多 6 个可核验披露行业，将组合占比映射到复苏、扩张、过热、放缓、收缩或证据不足；过热、放缓和收缩合计仅作为压力情景。经营生产力代理走弱的披露暴露单独显示。

压力暴露超过 10% 为关注，超过 20% 为优先复核。周期阶段不是择时或交易信号。

## 配置复核建议

建议由固定规则生成，优先级为 `info`、`watch` 或 `high`：

- 目标仓位全部填写且合计约为 100% 时，显示当前占比与目标偏差；绝对偏差达到 5 个百分点才提示复核；
- 目标仓位不完整时，只提示先补全目标，不猜测用户目标；
- 风险维度超过阈值时，给出“核对、检查、评估、人工决定是否分步再平衡”等复核动作；
- 证据缺失时，提示先核对日期和缺失项；
- 不输出具体买入/卖出对象、时点、金额、自动调仓比例、收益预测或收益承诺。

## 安全与失败语义

- 仅允许 localhost；
- 每次请求必须人工勾选确认；
- `readOnly=true`，账户、交易、通知、AI 和持久化能力必须全部为 `false`；
- Pydantic 与服务层都拒绝额外字段，金额或用户字段不能混入持仓快照；
- 组合占比合计必须在 100% 的 0.05 个百分点容差内；
- 固定 75 秒请求超时；
- Provider 异常只降级为固定缺失码，不透传 traceback、URL、Cookie、Header 或本机路径；
- 单项公开数据失败时保留其他维度，并明确 `partial` / `insufficient`；
- 测试只使用注入 Fake AKShare，不访问真实网络；
- 不读取、打印、上传或修改 `.env`、Token、Webhook、API key；
- 不接 Microsoft Qlib 依赖、真实账户、数据库、AI、通知或自动交易；
- 不修改 Electron safeStorage / Windows DPAPI、更新或 Release 链路。

## 回滚

回滚 D4 时只移除 `src/fund_portfolio_advice.py`、D4 只读 endpoint、`FundPortfolioAdvicePanel`、相关 API 类型、测试和本文档。D1–D3 基金公开资料、净值、比较、行业穿透、周期/生产力证据，以及股票中心、用户分域和 Windows Portable-M1 均不受影响。
