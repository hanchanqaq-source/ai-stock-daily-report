# Web-P17 账户首页综合看板说明

## 1. 设计目标

Web-P17 将账户概览、股票 ETF、场外基金净值、个人观察点位、风险提醒整合为本地 Web 首页综合看板。页面只做展示增强，继续保留轻量静态 HTML / CSS / JS 实现，不引入 React、Vue、Next.js 或外部样式库。

## 2. 数据来源

当前本地预览只读取 `web/static/demo_final_page_payload.json`。未来正式数据必须来自 P5-T `final_page_payload`，也就是最终安全闸门输出的页面 payload。

页面不直接读取 provider 原始数据，不请求真实行情，不直接请求原始行情字段，不请求真实基金净值，不直接请求原始基金净值字段，不读取真实 `user_config`。

## 3. 首页区域

首页综合看板包含：

- 账户总览：账户名称、`payload_status`、`display_mode`、`data_mode`、是否可写入仓库和数据安全状态。
- 安全状态：已审计、默认脱敏、禁止写入真实数据、仅作为个人观察和记录，需用户自行判断。
- 数据概览：股票 ETF 数量、场外基金数量、个人观察点位数量、blocked 数量、unavailable 数量和 redacted 数量。
- 股票 ETF 快览：复用 Web-P15 股票 / ETF 卡片渲染逻辑。
- 场外基金快览：复用 Web-P15 场外基金净值卡片渲染逻辑。
- 个人观察点位快览：复用 Web-P16 个人观察点位卡片渲染逻辑。
- 风险提醒：展示 payload warnings；为空时展示“暂无额外风险提示。”。
- 数据说明：说明页面只用于本地预览，不构成交易建议。

## 4. 安全边界

- 不请求真实行情，不直接请求原始行情字段。
- 不请求真实基金净值，不直接请求原始基金净值字段。
- 不请求原始行情字段。
- 不请求原始基金净值字段。
- 不读取真实 `user_config`。
- 不保存个人敏感字段 / 成本字段 / 账户字段。
- 不保存 Token / API Key / Webhook。
- 默认展示脱敏结果。
- 页面只消费 demo `final_page_payload` / 未来 P5-T `final_page_payload`。

## 5. 场外基金说明

场外基金不支持真正实时价格。页面只能展示净值、估算净值、净值日期、估算更新时间等净值相关字段，不把场外基金称为实时涨跌。

盘中估算仅供观察，最终以基金公司公布净值为准。

## 6. 个人观察标签

页面允许买入观察、加仓观察、减仓观察、止盈观察、止损观察、风险位等个人标签。这些标签仅用于个人观察和记录。

页面禁止必须买入、必须卖出、保证收益、无风险、自动下单、系统替你操作等表达。

页面必须保留：仅作为个人观察和记录，需用户自行判断。

## 7. 后续路线

- Web-P18：指数模块页面切换。
- Web-P19：清理中心页面。
- Web-P20：设置与导入导出页面。

账户首页可以使用“资金看板”“仓位观察”“成本观察”“盈亏观察”等个人工作台文案。股票 / ETF / 指数区域可以使用“实时涨跌”，场外基金区域仍必须使用净值 / 估算净值 / 最终以基金公司公布净值为准等表达。public demo 只展示脱敏值，不展示真实金额、成本价或账户资产。详见 `docs/personal_wording_policy.md`。

## 8. Web-P18 全球市场指数模块

Web-P18 在账户首页综合看板下方新增“全球市场指数”区域，支持 全球总览 / A股 / 港股 / 美股 / 韩股 tab 切换。该区域继续只读取本地 demo `final_page_payload` 中的 `market_indices` 或 fallback payload，不请求真实行情，不保存真实价格、涨跌幅、成交额或账户数据，并通过 badge 区分官方指数和系统计算指标。


## Web-P19 清理中心入口

账户首页旁新增清理中心 demo 区域，用于展示可清理 / 谨慎清理 / 禁止清理分类。该区域默认只扫描、不删除，必须先预览再确认；当前不删除真实文件，不清理 data/history、data/user_config、真实账户配置、真实金额、成本价、账户资产或 Token / API Key / Webhook。
