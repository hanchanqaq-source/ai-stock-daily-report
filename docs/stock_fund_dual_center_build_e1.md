# 股票基金双中心 Build E1：本地数据库持久化

## 结果

Build E1 将工作台用户、股票快速持仓和基金快速持仓写入项目已有 SQLite 数据库。刷新 Web 页面或重启便携程序后，数据会从本机数据库恢复；不需要安装数据库服务。

## 数据边界

| 数据 | 表 | 隔离方式 |
| --- | --- | --- |
| 工作台用户 | `workspace_users` | `self` 是不可删除的默认用户 |
| 股票快速持仓 | `workspace_stock_holdings` | 通过 `user_id` 归属用户 |
| 基金快速持仓 | `workspace_fund_holdings` | 通过 `user_id` 归属用户 |

这些表只保存用户手动录入的工作台数据，不代表登录账户，不读取券商/基金平台账户，也不与高级股票账户、交易流水或自动交易模型合并。删除非默认用户时，其两类快速持仓一并删除。

## 本机接口

`/api/v1/workspace-portfolio` 提供状态读取，以及用户、股票持仓、基金持仓的新增/重命名/删除操作。接口只接受 `127.0.0.1`、`::1` 和测试客户端请求。

## 界面完整显示

股票中心和基金中心继续作为两个明确入口。桌面侧栏改为内部纵向滚动，在窗口高度较小或系统缩放较大时仍可访问全部中心功能、设置和用户管理，不通过删除或隐藏功能解决溢出。

## 迁移与回滚

- 首次访问时自动创建默认“本人”档案；SQLAlchemy `create_all` 幂等创建三张新表。
- Build E1 以前的数据只存在页面内存，旧版本重启后没有可迁移的数据源，因此不会伪造历史持仓。
- 回滚代码不会主动删除三张表，已写数据保留；重新升级后可继续读取。
- 数据库继续使用项目既有数据目录和 SQLite 安全/并发配置，不新增配置项。
