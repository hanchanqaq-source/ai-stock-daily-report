# Windows Portable-M2 更新规划

## 当前阶段：M2.1（只校验）

Portable-M2.1 仅允许用户手动选择 Portable ZIP 与对应的 `.sha256` 文件，应用只读计算 ZIP 的 SHA-256，并检查 ZIP 内的中文 EXE、打包后的 backend EXE 与 `data/logs/config/plugins` 目录后显示“通过”或“不通过”。

- 不下载文件。
- 不解压、替换、覆盖、删除或重启程序。
- 不修改 `data/`、`config/`、`logs/`、`plugins/` 或任何持仓数据。
- 仅便携版在设置页显示“校验便携更新包”；安装版的检查/安装更新按钮不调用此能力。

结构检查只读取 ZIP 的中央目录，不解压到用户目录，不构成更新安装。

## M2.2A：恢复点基础（已实现，尚未连接替换入口）

已实现可测试的恢复点模块：只备份可替换的程序文件，并明确排除 `data/config/logs/plugins`；恢复时同样不触碰这些用户目录。恢复点只能位于程序目录内的 `.portable-update-backups/`，避免把用户数据写到未知位置。

本阶段没有解压、替换、删除、重启或对用户可见的安装按钮。

## M2.2B：替换与立即回退核心（已实现，尚未连接运行中 EXE 的重启交接）

已实现：对已经临时解压并验证过的程序目录，先创建恢复点，再仅替换程序文件；`data/config/logs/plugins` 不会复制、删除或覆盖。替换过程异常时立即由恢复点回退程序文件。

当前尚未从界面调用该模块，也不会尝试替换正在运行的 EXE；下一小版将通过退出后的受控 helper 完成重启交接与启动验证。

## M2.2C：用户确认后的受控重启交接（已实现）

仅 Windows 便携版在设置页提供独立的“备份并安全更新”入口；安装版的检查/安装更新按钮不调用它。用户仍需手动选择 ZIP 与 `.sha256`，且必须在系统确认框中再次确认后才会开始。

确认后，应用先在程序目录 `.portable-update-backups/` 创建只含程序文件的恢复点；退出后的 PowerShell helper 会再次计算 ZIP SHA-256、临时解压并复验中文 EXE、backend EXE 与 `data/config/logs/plugins`。通过后才仅替换程序文件并重启；若新进程在 8 秒启动验证期内退出，或任一步骤失败，则 helper 自动回退程序文件并重启旧程序。

恢复点、替换与回退始终排除 `data/config/logs/plugins`，不会覆盖持仓数据。该验证仅确认进程没有在启动期内退出，不等同于完整业务健康检查。
