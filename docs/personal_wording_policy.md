# 个人使用场景文案策略说明

## 1. 设计目标

本策略面向用户自己的个人投资观察工作台，不面向公开投顾产品。目标是在不放宽交易、安全和隐私边界的前提下，允许页面标题、页面说明和个人标签使用更直观、易懂的表达。

文案策略从简单关键词硬拦截改为上下文识别：个人观察标签可以更直接；股票、ETF、指数可以使用“实时涨跌”等行情描述；场外基金仍必须使用“估算涨跌 / 净值涨跌 / 估算净值 / 日净值”等净值语义。

## 2. 允许词语

个人观察和本地工作台上下文中允许以下直观表达：

- 买入观察
- 加仓观察
- 减仓观察
- 卖出观察
- 清仓观察
- 止盈观察
- 止损观察
- 持有观察
- 关注观察
- 风险观察
- 实时涨跌
- 股票实时涨跌
- ETF 实时涨跌
- 指数实时涨跌
- A股实时涨跌
- 港股实时涨跌
- 美股实时涨跌
- 韩股实时涨跌
- 实时行情
- 最新价
- 涨跌幅
- 涨跌额
- 资金看板
- 账户首页资金看板
- 仓位看板
- 成本看板
- 盈亏看板
- 实时涨跌看板
- 行情看板
- 仓位观察
- 成本观察
- 盈亏观察

这些表达允许出现，不代表系统提供买卖建议，也不代表系统可以自动交易。相关页面必须保留免责声明：仅作为个人观察，不构成交易建议，不自动下单。

## 3. 禁止词语

以下表达仍必须阻断：

- 必须买入
- 必须卖出
- 立刻买入
- 立刻卖出
- 无脑买入
- 无脑卖出
- 全仓买入
- 梭哈
- 自动下单
- 自动买入
- 自动卖出
- 替我买入
- 替我卖出
- 强制交易
- 保证收益
- 必赚
- 稳赚
- 一定赚钱
- 绝对上涨
- 必涨
- 翻倍保证
- 无风险收益

public demo、CI fixture 和仓库示例中还必须阻断真实隐私值和 secret 泄露，包括真实金额、成本价、账户资产、持仓金额、Token、API Key、Webhook、Secret、密码、身份证、手机号等。

## 4. 实时涨跌规则

“实时涨跌”不再作为全局禁词，而是按资产类型和语义上下文判断。

允许：

- 股票实时涨跌
- ETF 实时涨跌
- 指数实时涨跌
- A股实时涨跌
- 港股实时涨跌
- 美股实时涨跌
- 韩股实时涨跌

场外基金不能写成：

- 场外基金实时涨跌
- 普通基金实时涨跌
- 基金实时价格
- 场外基金实时价格
- 实时净值

场外基金应该写：

- 场外基金估算涨跌
- 基金净值涨跌
- 估算净值
- 日净值
- 最终以基金公司公布净值为准

## 5. 资金看板规则

本地个人工作台可以使用“资金看板”“账户首页资金看板”“仓位看板”“成本观察”“盈亏观察”等页面文案。

public repo、demo payload、CI fixture 和文档示例只能展示 redacted 或示例值，不能展示真实金额、真实成本价、真实账户资产，也不能把真实用户配置提交到仓库。本地个人模式以后可以展示真实值，但必须只存在本地，不进入 public repo。

## 6. 安全边界

词语放宽不代表自动交易放宽。系统仍不做自动下单、不替用户买入或卖出、不生成强制交易指令，也不承诺收益。

词语放宽不代表真实金额、成本价、账户资产、Token、API Key、Webhook 或其他 secret 可以进入仓库。本次策略只调整个人使用场景文案识别，不接真实行情、不接真实基金净值、不接真实账户配置，不修改 Discord、日报或周报主流程。
