# 全球市场指数矩阵与页面切换模型说明

## 1. 设计目标

本模块用于定义全球市场指数页面结构、市场切换模型、指数分组，以及官方指数和系统计算指标的分类。A股指数矩阵与体感指标见 `docs/cn_market_indicators.md`。A股中位数涨跌幅、上涨家数占比等属于系统计算指标，非官方指数。韩股指数矩阵与体感指标见 `docs/kr_market_indicators.md`。韩股中位数涨跌幅、上涨家数占比等属于系统计算指标，非官方指数；韩股体感指标不照搬 A 股涨跌停差逻辑。
当前阶段只输出 `model_only` 结构，不负责真实行情抓取，不联网，不接入 AKShare、yfinance、东方财富、Yahoo Finance、TradingView 或其他数据源，也不保存真实价格、涨跌幅、成交额、账户资产或密钥。

## 2. 市场页面

支持的页面 key 为：

- `global`：全球总览。
- `cn`：A股指数页面。
- `hk`：港股指数页面。
- `us`：美股指数页面。
- `kr`：韩股指数页面。

`unknown` 表示未知市场，默认不展示指数矩阵。

## 3. 页面切换规则

默认页面为 `global`。用户可以在全球总览、A股、港股、美股、韩股之间切换。页面 tab 顺序固定为：全球总览、A股、港股、美股、韩股。

## 4. 指数分组

市场指数矩阵按以下分组组织：

- 权重核心：用于观察市场核心权重资产或大盘代表指数。
- 成长科技：用于观察成长、科技或创新板块代表指数。
- 中小盘：用于观察中盘、小盘或更广市场覆盖指数。
- 市场体感：用于承载后续系统计算的广度、等权、平均或中位数类指标。

## 5. 官方指数 vs 系统计算指标

`official_index` 表示公开指数项，当前阶段只定义结构和后续数据源需求，默认 `source_status=pending_data_source`。

`computed_indicator` 表示系统后续根据市场数据计算的体感指标，非官方指数，默认 `source_status=computed_later`。所有系统计算指标必须在描述或免责声明中标注“非官方指数”或等价说明，并且不得被展示为官方指数。

## 6. A股指数矩阵

- 权重核心：上证指数、深证成指、沪深300、中证A500、上证50。
- 中小盘：中证500、中证1000、中证2000、国证2000。
- 成长科技：创业板指、创业板50、科创50、科创100、北证50。
- 市场体感：A股中位数涨跌幅、A股平均涨跌幅、A股上涨家数占比、A股涨跌停差、全A等权涨跌、A股成交额变化。

市场体感指标均为 `computed_indicator`，不是官方指数。

## 7. 港股指数矩阵

- 权重核心：恒生指数、恒生中国企业指数、恒生综合指数。
- 成长科技：恒生科技指数。
- 市场体感：港股中位数涨跌幅、港股上涨家数占比、港股平均涨跌幅、港股成交额变化。

港股体感指标均为 `computed_indicator`，不是官方指数。

## 8. 美股指数矩阵

- 权重核心：标普500、道琼斯工业平均指数、纳斯达克综合指数。
- 科技成长：纳斯达克100、费城半导体指数。
- 中小盘：罗素2000、标普400、标普600。
- 广度和体感指标：标普500等权指数、NYSE上涨家数占比、Nasdaq上涨家数占比、美股中位数涨跌幅、美股平均涨跌幅。

标普500等权指数作为公开指数项；NYSE / Nasdaq 上涨家数占比、美股中位数涨跌幅、美股平均涨跌幅和美股全市场等权涨跌属于 `computed_indicator`，不是官方指数。美股指数矩阵与广度指标见 `docs/us_market_indicators.md`；美股体感指标不照搬 A 股涨跌停差逻辑。

## 9. 韩股指数矩阵

- 权重核心：KOSPI、KOSPI 200、KRX 300。
- 成长科技：KOSDAQ、KOSDAQ 150。
- 市场体感：韩股中位数涨跌幅、韩股平均涨跌幅、韩股上涨家数占比、韩股全市场等权涨跌、韩股成交额变化。

韩股体感指标均为 `computed_indicator`，不是官方指数。韩股体感指标不照搬 A 股涨跌停差逻辑。
