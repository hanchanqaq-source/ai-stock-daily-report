# Web-P18 全球市场指数模块说明

## 1. 设计目标

Web-P18 在本地静态 Web 页面新增全球市场指数模块，支持全球总览、A股、港股、美股、韩股切换。该模块延续 Web-P17 的本地安全预览边界：只读取 `web/static/demo_final_page_payload.json` 或 `app.js` fallback payload，不引入 React / Vue / Next.js，不接真实 provider，不联网。

## 2. 页面切换

页面新增 `全球市场指数` 区域，包含以下 tab：

- 全球总览
- A股
- 港股
- 美股
- 韩股

默认显示 `全球总览`。用户点击 A股、港股、美股或韩股 tab 后，只切换本地 DOM 中对应的 demo 指数矩阵，不请求外部接口。

## 3. 指数矩阵

各市场展示的 demo 指数矩阵如下：

- 全球总览：A股摘要、港股摘要、美股摘要、韩股摘要、全球风险提示、数据说明。
- A股：权重核心、中小盘、成长科技、市场体感。
- 港股：权重核心、内地权重、科技成长、市场体感。
- 美股：权重核心、科技成长、中小盘、市场广度 / 体感。
- 韩股：权重核心、成长科技、综合指数、市场体感。

所有展示值均为 demo / redacted，不包含真实价格、真实涨跌幅、真实成交额、真实净值或真实账户金额。

## 4. 官方指数 vs 系统计算指标

官方指数是公开指数，例如沪深300、恒生指数、标普500、KOSPI 等。页面通过 `official_index` 类型和“官方指数” badge 展示。

系统计算指标是系统根据市场数据计算的体感指标，例如中位数涨跌、上涨家数占比、成交额观察、等权涨跌等。系统计算指标不是官方指数，页面通过 `computed_breadth_indicator` 或 `computed_sentiment_indicator` 类型和“系统计算指标” badge 展示，并明确提示：系统计算指标，非官方指数。

`demo_only` 用于当前只作为示例的摘要、风险提示或数据说明。

## 5. 安全边界

当前模块只展示 demo/redacted：

- 不请求真实行情。
- 不请求真实 provider。
- 不保存真实价格。
- 不保存真实涨跌幅。
- 不保存真实成交额。
- 不保存真实净值。
- 不保存真实账户数据。
- 不保存 Token / API Key / Webhook。
- 不连接 Discord / 日报 / 周报。
- 页面仅作为个人观察和记录，需用户自行判断。

`market_indices.can_write_to_public_repo` 必须为 `false`，`data_mode` 必须为 `dry_run`，`display_mode` 必须为 `redacted`。

## 6. 后续路线

- 后续可以接入真实指数行情 provider，但必须先通过安全闸门和 display adapter。
- 后续可以接入 A股 / 港股 / 美股 / 韩股市场广度统计。
- 后续可以接入 Web-P19 清理中心。
- 后续可以接入 Web-P20 设置与导入导出页面。
