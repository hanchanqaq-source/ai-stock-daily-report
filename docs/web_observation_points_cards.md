# Web-P16 个人观察点位卡片页面说明

## 1. 设计目标

Web-P16 将 `final_page_payload` 中的 `sections.observation_points` 渲染成个人观察卡片列表，让标签、关联资产、状态、风险等级和免责声明更清晰。页面仍保持轻量静态 HTML / CSS / JS，不引入 React、Vue、Next.js 或外部样式库。

## 2. 数据来源

当前本地预览只读取 `web/static/demo_final_page_payload.json`。

未来正式数据必须来自 P5-T `final_page_payload`，也就是 `src/account_real_data_final_gate.py` 输出的最终安全 payload。Web 页面不得绕过最终安全闸门直接消费 provider 原始数据、真实账户配置或未脱敏中间结果。

## 3. 允许的个人观察标签

允许展示的个人观察标签包括：买入观察、加仓观察、减仓观察、止盈观察、止损观察、清仓观察、低吸区、目标区、风险位、等待回调、继续持有、暂不操作。

扩展观察语义可使用：分批买入、个人观察、风险提醒、下一步关注。

## 4. 禁止表达

页面和 demo payload 禁止出现强制交易、收益承诺或自动执行表达，包括：必须买入、必须卖出、立即满仓、稳赚、保证收益、无风险、自动下单、系统替你操作。

同类禁止表达还包括：系统建议你买入、系统建议你卖出。

## 5. 页面字段

个人观察卡片展示字段包括：标签、关联资产、代码、类型、市场、观察点位、状态、风险等级、说明、免责声明。

页面也展示数据状态和“是否自动操作：否”，用于明确这些卡片只做个人观察记录。

## 6. 安全边界

- 不请求原始行情字段。
- 不请求原始基金净值字段。
- 不读取真实 `user_config`。
- 不保存个人敏感字段 / 成本字段 / 账户字段。
- 不保存 Token / API Key / Webhook。
- 不接 Discord、日报、周报或真实 provider。
- 观察点位默认脱敏，`point_display` 默认使用 `<redacted>`。

## 7. 免责声明

仅作为个人观察和记录，不自动下单，不构成强制交易指令。

- Web-P19：清理中心页面。
