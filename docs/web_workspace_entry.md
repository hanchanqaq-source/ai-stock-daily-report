# Web 工作台入口规范

Web 工作台是本地个人工作台，用于后续查看账户、股票、ETF、基金、观察点位和风险提示。Web-P0 只定义入口规范、目录骨架和文档边界，不实现真实页面，不引入 React、Vue、Next.js 或其他重型前端框架。

## 安全入口

Web 工作台默认只消费 `src/account_real_data_final_gate.py` 输出的 `final_page_payload`。该 payload 必须已经通过 P5-T 最终安全闸门，并且面向页面展示时处于 redacted / safe 状态。

Web 页面不得直接读取 provider 原始结果，不得直接请求真实行情，不得直接请求真实基金净值，不得直接读取真实 `user_config`。Web 页面不得保存 Token / API Key / Webhook，不得保存真实金额 / 成本价 / 账户资产。

blocked payload 只能显示安全错误提示或 blocked 状态，不得显示被拦截的真实值。

## 产品方向记录

后续 Web 工作台可以参考以下方向，但本次不复制外部产品、不实现真实 UI：

1. 左侧导航栏。
2. 顶部搜索框。
3. 中间主内容卡片。
4. 账户 / 持仓 / 观察 / 回测 / 设置 / 主题等入口。
5. 卡片可显示关联板块、个人观察点位、相关资讯、风险提示。
6. 页面文字可以更直接，但必须保持个人观察语气。
7. 禁止强制交易、自动下单和保证收益表达。

## 个人观察标签

页面允许使用个人观察表达：买入观察、加仓观察、减仓观察、止盈观察、止损观察、清仓观察、低吸区、目标区、风险位、等待回调、继续持有、暂不操作。

这些标签只能表示个人观察和记录，不代表系统替用户执行交易。

## 必备免责声明

本页面仅作为个人观察和记录，不自动下单，不构成强制交易指令。
