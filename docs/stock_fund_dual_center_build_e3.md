# 股票基金双中心 Build E3：快速持仓编辑

## 范围

Build E3 让当前用户可以在股票中心或基金中心直接编辑各自的“快速持仓”。股票编辑数量、平均成本、证券账户和备注；基金编辑金额、收益、目标仓位和备注。股票与基金依旧分区，且只能编辑当前用户自己的持仓。

## 保存规则

1. 编辑抽屉会先显示当前保存的字段。
2. 提交时由本机 `127.0.0.1` API 校验代码/名称、金额/数量、成本和目标仓位。
3. API 成功写入 SQLite 后，页面才替换该条记录；失败时保留旧持仓并给出提示。
4. 持仓 ID 和所属用户不可通过编辑接口改变；其他用户、另一资产类别或不存在的持仓会被拒绝。

## 非范围

- 不编辑高级股票账户、交易、资金流水、券商 CSV 或真实账户；
- 不接行情、AI、Provider、通知或自动交易；
- 不导出密钥；Build E2 的备份/恢复仍只处理快速持仓工作台。

## 验收

- 股票与基金各自显示“编辑”，并始终在对应中心打开；
- 成功保存后刷新/重启仍保留修改；
- 跨用户更新返回 404，失败保存不替换页面旧值；
- 后端 API 与 Web 构建测试通过。
