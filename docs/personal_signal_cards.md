# 个人观察点位卡片说明

## 1. 设计目标

个人观察点位卡片用于为后续 Web 页面提供结构化展示模型，帮助页面呈现用户自己的观察区、参考区和记录入口。本模块只定义模型、文案边界和 Markdown Demo，不负责真实行情计算，不接真实 provider，不读取真实 `user_config`。

## 2. 卡片类型

当前固定支持四类卡片：

- 观察买入区：展示个人记录买入的观察区间。
- 二次关注区：展示回调后的二次观察区或个人记录加仓参考区。
- 风险控制位：展示风险升高区间，并保留个人记录减仓或标记清仓入口。
- 目标观察区：展示目标观察区，并保留个人记录卖出或部分止盈记录入口。

## 3. 个人记录动作

允许的记录动作包括：记录买入、记录卖出、记录加仓、记录减仓、标记清仓。这些动作只用于更新用户自己的个人记录或资产状态，不代表系统自动执行任何操作。

## 4. 文案安全规则

卡片文案可以记录用户动作，不能主动指挥用户动作。展示语气应保持为“个人观察”“个人记录”“可作为记录参考”。不得使用建议、推荐、必须、立即等词语与交易动作组成指挥性表达。

页面必须保留免责声明：本卡片仅用于个人观察和记录，不会自动执行任何操作。

## 5. 数据限制

本阶段不接真实行情、不联网、不保存真实价格、不保存真实涨跌幅、不保存真实净值、不保存真实估算净值、不保存真实成交额，也不读取真实持仓或真实账户配置。`data_mode` 仅允许 `model_only` 或 `fixture_only`，`has_real_market_data` 必须为 `false`。

## 6. 后续路线

后续 Web-P16 可把该模型渲染为四宫格卡片，并与账户页面的个人记录入口联动。接入真实数据前仍需经过 provider 安全审查，并继续保证点位卡片只表达个人观察和记录语义。
