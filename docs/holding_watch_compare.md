# 持有 vs 收藏对比分析说明

## 1. 设计目标

持有 vs 收藏对比分析用于比较账户 / 分组内当前持有资产和收藏 / 观察资产的数量、标签方向和风险观察项。该能力面向后续工作台看板复用，本阶段只提供只读分析逻辑、结构化结果、Markdown 示例、文档和测试。

该模块不接入日报 / 周报主流程，不接入 Discord，不读取真实私人配置，不保存真实金额、成本价或账户资产，也不修改资产状态。

## 2. 状态范围

- `holding`：参与持有方向对比。
- `watching`：参与收藏 / 观察方向对比。
- `cleared`：只统计历史数量，默认不参与当前标签对比。
- `archived`：默认不参与当前对比。
- `deleted`：默认不参与当前对比。

当前 active 对比只比较 `holding` 和 `watching`。如果两类资产都为空，结果状态为 `empty`；如果只有其中一类，结果状态为 `insufficient_data`，并返回可读的数据不足说明。

## 3. 对比内容

结构化结果包括：

- 资产数量：分别统计 `holding`、`watching`、`cleared`、`archived`、`deleted`。
- 标签分布：按资产 `tags` 统计持有方向和收藏方向的标签频次。
- 共同标签：同时出现在持有和收藏中的标签。
- 仅持有标签：只出现在持有资产中的标签。
- 仅收藏标签：只出现在收藏资产中的标签。
- 风险观察：仅复用调用方传入的已有结构化风险信号，不新增数据源，不编造风险。
- 数据不足说明：当持有或收藏为空时，明确说明当前缺少哪一侧数据。

## 4. 输出规则

输出只能写观察，不写交易建议。Markdown 示例固定标注为 Demo，只展示账户名称、资产数量、标签、观察点、差异观察、风险与数据说明。

禁止输出或保存买卖指令、真实金额、成本价、账户资产、Webhook、Token、API Key、个人邮箱、手机号或身份证等敏感信息。

## 5. 后续网页工作台关系

后续 Web-P10 会使用该模块实现“持有 vs 收藏对比看板”。网页侧可以直接复用结构化结果中的 `asset_counts`、`holding`、`watching`、`comparison` 和 `data_warnings` 字段进行展示。

本阶段不实现真实网页页面，也不接入正式日报、周报或 Discord 推送。

## 6. 安全规则

public 阶段不保存金额、成本价、账户资产。模块只读取传入的账户 / 分组对象，输出公开安全字段，不修改原始 group，不修改资产状态，不执行清仓、归档、删除或任何持久化操作。
