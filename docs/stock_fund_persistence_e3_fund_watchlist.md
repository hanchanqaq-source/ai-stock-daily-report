# 持久化阶段 E3：基金自选

## 目标

为每个工作台用户建立独立的基金自选列表，并写入 Build E2 已确认的唯一正式 SQLite 数据库。基金自选只表示“关注”，不是基金持仓，不需要金额、收益或目标仓位。

本阶段编号属于新的持久化路线，与历史文档“股票基金双中心 Build E3：快速持仓编辑”不是同一任务；旧功能保持不变。

## 正式链路

`FundWatchlistPanel → PortfolioUserContext → workspacePortfolioApi → /api/v1/workspace-portfolio → SQLAlchemy Session → workspace_fund_watchlist_items → DATABASE_PATH`

- 前端不直接打开 SQLite；
- 数据仍只进入 `DATABASE_PATH` 指向的 `stock_analysis.db`；
- 不创建第二个数据库或第二套用户表；
- 新增迁移记录仍写入既有 `schema_migrations`。

## 数据与校验

每条基金自选只保存：

- 本机生成的记录 ID；
- 所属工作台用户 ID；
- 六位基金代码；
- 用户手动输入的基金名称；
- 可选备注；
- 创建时间。

同一用户内基金代码唯一，不同用户可分别关注同一基金。同一代码也可以同时存在于基金自选和基金持仓；加入、编辑或移出自选均不得修改持仓。

## 页面行为

- 基金中心新增 `/funds/watchlist` 和“基金自选”侧栏入口；
- 当前用户可手动新增、编辑名称/备注、两步确认移出；
- 切换用户后只显示该用户的基金自选；
- 保存失败后沿用 Build E2 的服务端回读与错误状态；
- 页面不自动联网补全基金信息，不读取账户，不调用 AI，不通知，不交易。

## 重启、备份与兼容

- 重建后端应用和数据库会话后，基金自选仍从同一个 SQLite 恢复；
- 删除普通用户时，同时删除该用户的基金自选；
- 工作台 JSON 备份白名单增加 `fund_watchlist_by_user`；
- 旧版 version 1 备份没有该字段时按空自选导入，保持向后兼容；
- 导入前预览和恢复点继续遵守明确确认及 localhost 边界。

## 本阶段不做

- 不在“问基金”或基金分析页自动选择自选基金；
- 不实现“手动输入 / 我的持仓 / 我的自选”统一选择器；
- 不自动把公开查询结果加入自选；
- 不接真实基金账户、自动交易、通知或 AI。

其中“手动输入 / 我的持仓 / 我的自选”统一选择器及既有只读页面接入已由[持久化阶段 E4](stock_fund_persistence_e4_fund_analysis_sources.md)承接。基金问答会话、分析记录持久化及从公开查询结果显式加入自选仍属于后续阶段。
