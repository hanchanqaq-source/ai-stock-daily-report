# 持久化阶段 E4：基金分析来源统一

## 目标

在基金中心建立统一的分析对象选择方式，让用户可以从以下三种来源准备基金代码：

1. 手动输入；
2. 当前用户的基金持仓；
3. 当前用户的基金自选。

本阶段编号属于新的持久化路线，与历史文档“股票基金双中心 Build E4：快速持仓回收站”不是同一任务；旧回收站功能保持不变。

## 正式链路

`FundAnalysisSourceSelector → PortfolioUserContext → 当前用户 fund holdings / fund watchlist → 既有只读基金页面`

- 选择器只消费 `PortfolioUserContext` 已加载的当前用户状态；
- 不直接请求数据库，不新增 API，不新增表，不创建第二套持久化；
- 手动输入和页面临时选择只保存在当前 React 页面内存中；
- 切换工作台用户时，对应页面以用户 ID 重新创建，清空上一用户的临时代码、确认状态和查询结果；
- 持仓金额、收益、目标仓位和自选备注不会进入公开基金查询请求。

## 页面接入

| 页面 | 允许数量 | E4 行为 |
| --- | ---: | --- |
| 基金首页公开资料 | 1 | 手动输入、我的持仓或我的自选选定一只；仍需勾选只读确认并点击“手动读取” |
| 问基金 | 1–4 | 只准备基金代码上下文；本阶段不启动 AI、不读取公开数据、不保存会话 |
| 基金对比 | 2–4 | 三种来源统一选择；仍需人工确认并点击读取计算 |
| 行业持仓穿透 | 1–4 | 三种来源统一选择；仍需人工确认并点击读取计算 |
| 行业周期 | 1–4 | 三种来源统一选择；仍需人工确认并点击读取证据 |
| 基金仓位与风险建议 | 当前全部基金持仓 | 继续固定使用当前用户真实录入的快速持仓，不允许用自选替代持仓 |

同一来源列表按基金代码去重。多选达到页面上限后，不再允许追加；手动输入仍由页面原有六位代码、数量和重复校验负责。

## 用户隔离

- “我的持仓”和“我的自选”仅展示当前用户的数据；
- 不显示另一个工作台用户的基金名称或代码；
- 切换用户不会沿用上一用户的临时选择、公开查询结果或只读确认；
- 选择自选基金不会自动加入持仓，选择持仓基金也不会自动加入自选。

## 安全边界

- 选择代码不会自动联网；
- 公开数据页面仍要求逐次人工确认和明确按钮操作；
- 不读取证券或基金账户；
- 不调用 AI、不发送通知、不执行交易；
- 不保存本次分析来源、代码选择或公开查询结果；
- 不修改 E3 基金自选数据，不修改基金持仓数据；
- D1/D2/D3 的 localhost、只读、无持久化请求契约保持不变。

## 后续阶段

E4 只提供统一来源选择和既有只读页面接入。基金问答会话、基金分析记录持久化、从公开查询结果显式加入自选等能力需要单独设计和授权，不在本阶段自动开启。
