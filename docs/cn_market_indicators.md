# A股指数矩阵与体感指标说明

## 1. 设计目标

本模块用于在 P5-L0 全球市场指数矩阵基础上，细化 A股指数矩阵和体感指标计算框架。当前阶段仅处理调用方传入的样例结构，不联网、不抓真实行情、不接入 AKShare / 东方财富 / 同花顺 / TradingView，也不保存真实价格、真实涨跌幅或真实成交额。

## 2. A股官方指数矩阵

A股官方指数按以下分组组织：

- 权重核心：上证指数、深证成指、沪深300、中证A500、上证50。
- 中小盘：中证500、中证1000、中证2000、国证2000。
- 成长科技：创业板指、创业板50、科创50、科创100、北证50。

这些条目使用 `item_type=official_index`，只定义页面和数据源准备结构，本阶段不写入行情点位、涨跌幅或成交数据。

## 3. A股体感指标

A股市场体感分组包含以下系统计算指标：

- A股中位数涨跌幅。
- A股平均涨跌幅。
- A股上涨家数占比。
- A股涨跌停差。
- 全A等权涨跌。
- A股成交额变化。

这些条目使用 `item_type=computed_indicator`，并标注 `is_computed=true`、`is_official=false` 和“非官方指数”免责声明。

## 4. 官方指数 vs 系统计算指标

官方指数是公开指数，当前只作为指数矩阵中的稳定结构项，后续由正式数据源补充行情能力。

体感指标是系统根据输入股票数据计算的指标，非官方指数。它们用于描述市场广度或体感，不应被展示为官方指数，也不应输出买卖建议。

## 5. 计算口径

- 中位数使用所有有效 `change_pct` 的中位数。
- 平均涨跌使用所有有效 `change_pct` 的算术平均。
- 上涨家数占比使用 `change_pct > 0` 的数量除以有效 `change_pct` 数量。
- 涨跌停差 = 涨停家数 - 跌停家数。
- 全A等权涨跌当前使用有效股票涨跌幅的简单平均作为近似值，后续接入正式数据源后可优化口径。
- 成交额变化当前只定义结构；没有 `previous_turnover` 等可比输入时返回 `insufficient_data`。

`None`、空字符串和非数字 `change_pct` 会被忽略；没有有效数据时返回 `empty` 或 `insufficient_data` 状态。

## 6. 数据限制

本阶段不联网、不抓真实行情，只处理输入数据。模块不读取真实用户配置，不保存真实价格、真实涨跌幅、真实成交额、账户资产、Webhook、Token 或 API Key。
