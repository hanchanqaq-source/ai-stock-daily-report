# Web-P19 清理中心说明

## 1. 设计目标

Web-P19 清理中心用于在本地 Web 工作台展示哪些缓存、日志、临时报表可以清理，哪些核心数据禁止清理。当前页面只消费本地 demo `cleanup_center` payload 或前端 fallback，不联网、不接真实 provider、不读取真实 `user_config`，也不执行真实删除。

## 2. 清理原则

默认只扫描，不删除。

清理流程必须遵循：先预览，再确认，再执行。

本阶段只做 demo，不真实删除。即使页面展示“预览清理”或“需要确认”，也只用于说明清理前置步骤，不会删除任何文件。

## 3. 可清理项目

可清理项目仅作为 demo 展示，包括临时缓存、临时测试输出、demo 预览缓存、旧的前端构建缓存等。这些项目在真实功能落地前也必须先预览，再由用户确认。

## 4. 谨慎清理项目

谨慎清理项目包括旧报告副本、旧日志、过期 artifact 摘要、debug 临时输出等。这些内容可能影响历史对照或排障复盘，因此必须在本地预览影响范围，并要求用户确认；当前 Web-P19 不执行真实清理。

## 5. 禁止清理项目

禁止清理 data/history。

禁止清理 data/user_config。

禁止清理真实账户配置、真实金额、成本价、账户资产、真实持仓数据、真实成本价数据和真实账户资产数据。

禁止清理 `.env`，禁止输出或保存 Token / API Key / Webhook 的真实值，Token / API Key / Webhook 永远不能清理到 public repo 或输出。

禁止清理 `AGENTS.md`、`ENTRY.md`、`README.md`、`docs/product_rules.md`、`docs/ERRORS_AND_LESSONS.md` 等仓库规则、入口说明、产品规则和经验文档。

## 6. 安全边界

Web-P19 不联网，不调用真实清理接口，不删除真实文件，不保存 secrets，不扫描真实文件系统，不接 Discord / 日报 / 周报，不接真实 provider。

页面按钮只能用于 demo 预览提示，点击后只能提示“当前为 demo 模式，不会删除任何文件。”，不能触发真实删除、写文件或请求删除 API。

## 7. 后续路线

后续 Web-P20 可做设置与导入导出页面。

后续真正清理功能必须二次确认，并且只允许本地执行；真实清理功能需要明确预览清单、影响说明、用户确认、回滚建议和禁止清理清单。
