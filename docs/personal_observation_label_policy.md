# 个人观察标签与直接表达规则

## 1. 设计目标

账户页面是用户自己使用的个人投资观察工具，不是自动交易系统。页面允许使用更直接、更接近个人记录习惯的观察标签，例如“买入观察”“加仓观察”“减仓观察”“止盈观察”“止损观察”“清仓观察”等。

这些词不再作为绝对禁词处理，而是按上下文识别：当它们用于页面标签、卡片标题、个人记录入口或观察结论时，归类为 `personal_observation_label`；当它们与强制交易、保证收益或自动下单表达组合时，必须阻断。

## 2. 允许标签

允许作为页面标签 / 卡片标题 / 个人记录入口使用的个人观察标签包括：

- 买入观察
- 分批买入
- 低吸区
- 加仓观察
- 继续持有
- 等待回调
- 减仓观察
- 冲高减仓
- 止盈观察
- 止损观察
- 清仓观察
- 风险位
- 目标区
- 压力位
- 支撑位
- 暂不操作
- 仅记录
- 个人观察
- 个人计划
- 观察结论
- 风险提醒
- 下一步关注

也允许更短的卡片标题：

- 买入区
- 加仓区
- 持有区
- 减仓区
- 止盈区
- 止损区
- 清仓区
- 等待区
- 风险区
- 目标区

## 3. 禁止表达

以下表达仍然禁止，必须返回 blocked 或 high / blocker 级别：

- 必须买入
- 必须卖出
- 必须加仓
- 必须清仓
- 立即买入
- 立即卖出
- 立即满仓
- 马上满仓
- 梭哈
- 稳赚
- 保证收益
- 无风险
- 一定涨
- 一定翻倍
- 系统建议你买入
- 系统建议你卖出
- 系统替你下单
- 自动下单
- 自动买入
- 自动卖出
- 代替用户操作

## 4. 安全说明

允许标签只代表个人观察和记录，不代表系统自动交易，不构成强制交易指令。页面展示个人观察标签时，应同时展示免责声明：

> 本页面标签仅作为个人观察和记录，不自动下单，不构成强制交易指令。

该规则不改变真实数据安全边界：本模块不请求真实行情、不请求真实基金净值、不保存真实数据、不读取真实 `user_config`，也不替用户执行任何交易或账户操作。
