# App-M4.1 设置工作台骨架

App-M4.1 为“股票基金质量分析系统”的 React Web 应用在既有 Web 设置页内新增正式的“设置”工作台骨架，用于后续接口配置、数据源管理和连接测试能力的分阶段接入。它只是静态工作台骨架，不替换、不删除、不降级既有设置分类、配置保存、认证、定时任务、通知测试、AlphaSift 开关和导入导出等既有设置功能。

## 本阶段完成内容

- 左侧导航保留 `首页`、`问股`、`持仓`、`回测`、`设置`、`主题` 的主要入口顺序，`设置` 位于 `回测` 与 `主题` 之间。
- `/settings` 路由继续渲染既有设置页面，并在页面内追加 App-M4.1 静态工作台骨架；刷新或直接访问不会因为工作台骨架缺少后端配置而报错。
- 设置工作台提供三个页签：
  1. `接口与密钥`
  2. `数据源管理`
  3. `连接测试`
- 默认打开 `接口与密钥` 页签。
- 页面仅展示静态状态、禁用控件和安全边界提示。

## 三个设置页签的用途

### 接口与密钥

展示未来接口配置入口，包括：

- AI 模型接口
- 行情数据接口
- 通知接口

每张卡片仅展示接口名称、当前阶段状态、用途说明和禁用的配置控件，并明确标注 `App-M4.1 暂不保存真实密钥`。

### 数据源管理

展示数据源静态阶段状态：

- `AkShare 公开 A 股行情` 标记为公开只读行情。
- 当前阶段说明为已具备手动 dry-run 基础。
- 默认状态为关闭。
- 正式页面接入尚未启用。
- 明确无账户读取、无交易权限、无密钥要求。
- Longbridge、Tushare、账户接口和基金净值接口统一显示为后续接入。

### 连接测试

展示连接测试的界面骨架和固定本地说明：`App-M4.1 尚未启用真实连接测试。` 本阶段连接测试按钮保持禁用，不会展示成功态，不会发送任何请求。

## 当前明确没有实现什么

App-M4.1 不实现以下能力：

- 不读取、展示或修改 `.env`。
- 不保存真实 API Key、Token、Webhook 或账户信息。
- 不使用 localStorage、sessionStorage、IndexedDB 或后端接口持久化 App-M4.1 接口配置。
- 不初始化 Longbridge、Tushare、账户接口或基金净值接口 Provider。
- 不发起 fetch、axios、WebSocket、Python 后端或外部地址请求。
- 不连接账户，不读取账户，不写数据库。
- 不调用 AI、通知、交易或定时任务能力。
- 不伪造连接成功、延迟数值、实时行情或 API 可用状态。

## 为什么暂时不保存真实密钥

真实密钥保存需要先完成前后端配置契约、敏感字段脱敏、权限边界、审计日志、错误文案和回滚策略。App-M4.1 只负责建立可见页面和安全占位，不在缺少完整安全设计时引入任何凭证持久化路径。

## 为什么连接测试保持禁用或本地说明

真实连接测试会涉及网络请求、Provider 初始化、凭证读取、错误分类和结果脱敏。为了避免把静态工作台误认为可用连接能力，App-M4.1 仅保留按钮位置和固定本地说明，不展示 `已连接`、`测试成功` 或延迟等可能误导用户的状态。

## 后续 App-M4.2 边界

App-M4.2 尚未开始，仍需单独授权后才能设计和实现。后续阶段至少需要先明确：

- 哪些接口允许配置，哪些字段属于敏感字段。
- 密钥保存位置、加密或托管策略、权限和回滚方案。
- 连接测试允许访问的目标、超时、重试、错误脱敏和日志策略。
- 数据源 Provider 的默认关闭策略、只读边界和账户隔离策略。
- 对 Web、后端、桌面端和文档的兼容影响。
