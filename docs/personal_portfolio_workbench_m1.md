# 个人持仓工作台 M1

## 目标

把现有持仓页从“专业账本优先”调整为“个人持仓分析优先”。基金和股票分开展示、分开命名、分开管理：基金只保留基金持仓分析；股票继续使用现有账户、交易和券商工具，但统一归入股票区域和独立的“股票高级管理”页面。

## 页面结构

1. 我的持仓分析
   - 页面默认只强调持有金额、收益、仓位和风险
   - 基金和股票分开管理，不使用模糊的公共账本入口
2. 股票账户摘要
   - 股票账户总资产
   - 股票持仓市值
   - 股票可用现金
   - 股票浮动收益
   - M1 的现有快照数据来自股票账本，因此名称显式标注“股票”
3. 基金持仓分析
   - 参考个人基金持有页的卡片式信息层级
   - M1 不伪造基金数据
   - 显示基金持有金额、持有收益、收益率和当前仓位的占位结构
   - 不展示资金流水、公司行为、券商 CSV 或股票账户管理
4. 股票持仓分析
   - 股票账户、股票代码、持有数量、平均成本、当前价格、股票市值、浮动盈亏和收益率
   - 股票账户和股票成本口径控件放在股票区域内
   - 继续消费现有 Portfolio Snapshot，不改变现有股票数据契约
5. 股票高级管理
   - 从股票区域进入独立页面
   - 包含交易录入、资金流水、公司行为、券商 CSV、股票账户管理和精确成本核算
   - 提供“返回我的持仓分析”入口
   - 不再把这些股票工具作为整个持仓页的公共“高级账本”
6. 股票风险概览
   - 最大股票占比
   - 股票账户回撤
   - 股票止损提醒
   - 股票 AI 风险信号数量

## 边界

- 仅修改 Web 前端路由和页面。
- 不新增或修改后端 API。
- 不新增数据库字段或基金资产类型。
- 不删除既有股票账本功能。
- 不伪造基金持仓、收益或仓位数据。
- 不接券商账户、支付宝接口或真实基金 Provider。
- 不主动调用 AI、通知或交易能力。

## M2 计划

下一阶段新增基金与股票资产类型，以及“快速录入当前持仓”：

- 基金：名称/代码、持有金额、成本或收益、赛道、目标仓位和备注。
- 股票：名称/代码、持有数量、平均成本和备注。
- 基金使用卡片式展示，股票继续使用专业表格。
- 股票交易流水、资金流水、公司行为和券商 CSV 继续保留在“股票高级管理”。

- 股票持仓由现有快照真实映射。
- 股票账户和成本口径切换会重新加载对应快照与风险。
- 股票高级管理进入独立路由，并可返回“我的持仓分析”。
