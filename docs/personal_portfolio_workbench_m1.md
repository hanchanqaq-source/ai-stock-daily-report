# 个人持仓工作台 M1

## 目标

把现有持仓页从“专业账本优先”调整为“个人日常查看优先”。默认页面先展示资产、收益、股票持仓和风险，高频查看不再要求用户面对交易流水、资金流水、公司行为和券商 CSV 工具。

## 页面结构

1. 个人资产总览
   - 总资产
   - 股票持仓市值
   - 可用现金
   - 持仓浮动收益
2. 基金持仓卡片骨架
   - 参考个人基金持有页的卡片式信息层级
   - M1 不伪造基金数据
   - 显示基金持有金额、持有收益、收益率和当前仓位的占位结构
3. 股票持仓表格
   - 账户、代码、数量、平均成本、当前价格、市值、浮动盈亏和收益率
   - 继续消费现有 Portfolio Snapshot，不改变现有股票数据契约
4. 个人风险概览
   - 最大单项占比
   - 当前回撤
   - 止损提醒
   - AI 风险信号数量
5. 高级账本
   - 默认收起
   - 展开后加载既有完整持仓页
   - 保留账户管理、交易录入、资金流水、公司行为、券商 CSV、历史记录和精确成本核算

## 边界

- 仅修改 Web 前端路由和页面。
- 不新增或修改后端 API。
- 不新增数据库字段或资产类型。
- 不删除既有持仓功能。
- 不伪造基金持仓、收益或仓位数据。
- 不接券商账户、支付宝接口或真实基金 Provider。
- 不主动调用 AI、通知或交易能力。

## M2 计划

下一阶段新增基金与股票资产类型，以及“快速录入当前持仓”：

- 基金：名称/代码、持有金额、成本或收益、赛道、目标仓位和备注。
- 股票：名称/代码、持有数量、平均成本和备注。
- 基金使用卡片式展示，股票继续使用专业表格。
- 交易流水和资金流水继续作为可选的高级账本能力。

## 验证

Web 改动应执行：

```bash
cd apps/dsa-web
npm ci
npm run lint
npm run build
```

并覆盖：

- 默认只展示个人持仓工作台。
- 基金区域不展示虚构数据。
- 股票持仓由现有快照真实映射。
- 高级账本默认不加载，展开后才加载既有完整页面。
- 账户和成本口径切换会重新加载对应快照与风险。
