# 个人持仓工作台 M1 / M1.1

## 目标

把现有持仓页从“专业账本优先”调整为“个人持仓分析优先”。基金和股票分开展示、分开命名、分开管理；M1.1 再增加独立用户档案，让本人、家人或其他持仓主体可以在同一工作台中切换，但不把用户档案误做成网络登录账号。

## 页面结构

1. 我的持仓分析
   - 页面默认只强调持有金额、收益、仓位和风险
   - 基金和股票分开管理，不使用模糊的公共账本入口
   - 标题区域提供“当前用户”切换和“管理用户”入口
2. 用户管理
   - 左侧导航在“用量”和“主题”之间增加“用户”
   - 支持新增、切换、重命名和删除非默认用户
   - 默认用户为“本人”，不可删除
   - M1.1 只保存于当前前端运行内存，刷新页面后恢复为“本人”
   - 非“本人”用户暂时显示独立空持仓，避免误展示本人的股票快照
3. 证券账户摘要
   - 证券账户总资产
   - 股票持仓市值
   - 证券账户可用现金
   - 股票浮动收益
   - 现有快照数据来自股票账本，因此名称显式标注“证券账户”或“股票”
4. 基金持仓分析
   - 参考个人基金持有页的卡片式信息层级
   - M1 不伪造基金数据
   - 显示基金持有金额、持有收益、收益率和当前仓位的占位结构
   - 不展示资金流水、公司行为、券商 CSV 或证券账户管理
5. 股票持仓分析
   - 证券账户、股票代码、持有数量、平均成本、当前价格、股票市值、浮动盈亏和收益率
   - 证券账户和股票成本口径控件放在股票区域内
   - 继续消费现有 Portfolio Snapshot，不改变现有股票数据契约
6. 股票高级管理
   - 从股票区域进入独立页面
   - 包含交易录入、资金流水、公司行为、券商 CSV、证券账户管理和精确成本核算
   - 提供“返回我的持仓分析”入口
   - 不再把这些股票工具作为整个持仓页的公共“高级账本”
7. 股票风险概览
   - 最大股票占比
   - 证券账户回撤
   - 股票止损提醒
   - 股票 AI 风险信号数量

## 用户与证券账户的层级

```text
用户档案
  └─ 我的持仓
      ├─ 基金持仓
      └─ 股票持仓
          └─ 证券账户
```

用户是持仓主体；证券账户只是某个用户下面的股票账户。两者不能混用。

## 边界

- 仅修改 Web 前端路由、导航、上下文和页面。
- 不新增或修改后端 API。
- 不新增数据库字段、登录账号或基金资产类型。
- 不写入浏览器存储，不保存真实用户资料。
- 不删除既有股票账本功能。
- 不伪造基金持仓、收益或仓位数据。
- 不接券商账户、支付宝接口或真实基金 Provider。
- 不主动调用 AI、通知或交易能力。

## M2 计划

下一阶段新增基金与股票资产类型，以及按用户保存的“快速录入当前持仓”：

- 用户：后端持久化、默认用户、用户切换和数据隔离。
- 基金：名称/代码、持有金额、成本或收益、赛道、目标仓位和备注。
- 股票：名称/代码、持有数量、平均成本和备注。
- 基金使用卡片式展示，股票继续使用专业表格。
- 股票交易流水、资金流水、公司行为和券商 CSV 继续保留在“股票高级管理”。

- 持仓页可以切换当前用户。
- 非默认用户不会误显示“本人”的股票快照。

- 证券账户和成本口径切换会重新加载对应快照与风险。
