# 个人持仓工作台 M1 / M1.1 / M2.1

## 目标

把现有持仓页从“专业账本优先”调整为“个人持仓分析优先”。基金和股票分开展示、分开命名、分开管理；M1.1 增加独立用户档案；M2.1 再提供按当前用户录入基金和股票代码的可用入口，以及截图上传、预览和识别结果确认区。

## 页面结构

1. 我的持仓分析
   - 标题区域提供“当前用户”“添加持仓”“截图识别”和“管理用户”
   - 基金和股票分开管理
2. 用户管理
   - 左侧“用户”位于“用量”和“主题”之间
   - 支持新增、切换、重命名和删除非默认用户
   - 默认用户为“本人”，不可删除
3. M2.1 快速录入
   - 基金：代码或名称、持有金额、持有收益、目标仓位和备注
   - 股票：代码或名称、持有数量、平均成本、所属证券账户和备注
   - 录入前明确显示目标用户
   - 基金和股票条目按当前用户隔离
   - 支持删除误录条目
4. 截图识别入口
   - 支持选择 PNG、JPG、WEBP 截图并在本机预览
   - 页面提供“识别结果确认区”
   - M2.1 不伪造 OCR 结果，也不会自动写入持仓
   - M2.2 再接真实识别服务，识别后必须人工确认
5. 基金持仓分析
   - 快速录入后展示基金代码、名称、金额、收益和目标仓位
   - 汇总基金持有金额、持有收益和估算收益率
   - 不展示资金流水、公司行为或券商 CSV
6. 股票持仓分析
   - 快速录入股票单独展示代码、名称、数量、平均成本、所属证券账户和录入成本
   - “本人”继续消费现有 Portfolio Snapshot
   - 正式股票交易工具继续位于“股票高级管理”
7. 股票高级管理
   - 包含交易录入、资金流水、公司行为、券商 CSV、证券账户管理和精确成本核算
8. 股票风险概览
   - 最大股票占比、证券账户回撤、止损提醒和 AI 风险信号

## 用户与证券账户层级

```text
用户档案
  └─ 我的持仓
      ├─ 基金持仓
      └─ 股票持仓
          └─ 证券账户
```

用户是持仓主体；证券账户只是某个用户下面的股票账户。

## 边界

- 仅修改 Web 前端路由、导航、上下文、页面和测试。
- 不新增或修改后端 API、数据库或真实登录账号。
- M2.1 快速录入数据只存在当前前端运行内存，刷新或重启后清空。
- 截图仅在当前浏览器进程中预览，不上传。
- 不伪造截图识别结果，不自动写入持仓。
- 不接券商账户、支付宝接口、真实基金 Provider、AI、通知或交易能力。

## 后续计划

- M2.2：真实截图识别、结果校对和确认录入。
- M2.3：用户与持仓持久化。
- M2.4：行情、仓位、风险和合理位置分析。

## 验证

```bash
cd apps/dsa-web
npm ci
npm run lint
npm run test -- PersonalPortfolioPage.test.tsx QuickHoldingEntryDrawer.test.tsx
npm run build
```

应覆盖：

- 左侧用户入口与当前用户切换。
- 基金和股票快速录入只写入当前用户。
- 录入后页面显示代码、名称和关键数值。
- 截图入口显示本机预览和识别结果确认区。
- 截图不会自动生成或保存虚假识别结果。
- 非默认用户不会误显示“本人”的正式股票快照。
- 股票账本工具仍只属于股票高级管理。
